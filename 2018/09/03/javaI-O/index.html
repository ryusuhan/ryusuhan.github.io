컴퓨터 내부 또는 외부의 장치와 프로그램간의 데이터를 주고받는 것!

자바의 문자열 입출력

문자열을 사용자로부터 입력받아서 화면에 출력하는 함수로 C언어에서의 문자열 출력함수 printf(), 문자열 입력함수 scanf()를 자바에서 이용하는 방법!
키보드에서 사용자로 부터 키 입력을 받을 때는 System.in을 사용한다.

문자열 입력!

우선 scanf()에 해당하는 문자열 입력 방법 3가지를 알아보면

  • Scanner를 사용한다.
  • System.in.read()함수와 BufferedReader를 혼용하여 사용한다.

1. Scanner 객체를 사용할 때

위처럼 Scanner 객체를 생성하고 System.in을 연결하면 표준입력스트림을 받을 수 있다.

1-1. Enter 입력 예외처리

위에서 주의해야할 점이 있다.
문자열을 입력 받을 때 사용하는 함수는 s.next()와 s.nextline() 두개가 있었다. 그렇다면 두 개의 차이점은 무엇일까?

-next()는 공백 단위로 문자열을 읽는다.
-nextLine()은 Enter 단위로 문자열을 읽는다. (Enter 입력은 \r\n으로 구성된 두개의 문자이다.)

그렇기 때문에 next()와 nextLine()을 섞어서 사용할 때 주의 해야한다.

위와 같은 코드에서
next() 실행 라인에서 숫자 7과 Enter를 입력하면 nextLine()에서 입력을 기다리지 않고 바로 통과 되어 버린다.
이유는, nextInt()가 실행될 때 숫자 7만 읽고, Enter(\r\n)값은 입력 스트림 버퍼에 남아 있기 때문이다.
따라서, nextLine() 실행시에 남아 있는 Enter 값을 읽어들이고 통과하게 된다.

따라서, 원하는 대로 동작하려면 아래와 같이 작성해야 한다.

2. System.in.read()함수 사용

두 번째 방법은 System 클래스에서 제공하는 read()라는 함수를 이용하여 키보드 입력을 받는 것이다.
read()함수는 1byte 크기만 입력스트림에서 읽어와서 ASCII 코드에 해당하는 int 값을 리턴한다.

그래서 문자를 입력받을 때 주로 사용한다.

그렇다면 Enter에 대한 예외처리는 어떻게 해야하는가?

숫자 + Enter 입력 시, Enter(\r\n)값은 입력스트림 버퍼에 그대로 남아 있게 되므로 아래와 같이 처리해 주어야 한다.

3. BufferedReader 객체 사용

BufferedReader 객체는 문자를 스트림에서 읽을 때 버퍼를 사용하여 조금 더 문자열 처리를 편하게 해주는 객체이다.

InputStreamReader의 경우 한 문자씩 스트림에서 읽어 오지만, BufferedReader는 미리 스트림에 있는 데이터를 버퍼에 받아온다.

위와 같이 readLine()함수를 사용하면 \r\n을 인식하며 한 줄을 읽어온다. Scanner의 nextLine()함수와 유사하게 동작한다.

문자 하나만을 읽고 싶다면 System.in.read()함수를 사용한다.

Scanner의 경우 nextInt를 사용하면 공백 다음 숫자를 받기 때문에 어려움 없이 사용할 수 있지만 BufferedReader의 경우 다른 방식을 취해 주어야 한다.
그것이 바로바로 StringTokenizer!!

3-1. StringTokenizer

이름과 같이 String을 특정 구분자(delimeter) 값을 통해 분리하는 함수다. 기본적으로는 공백을 기준으로 문자열을 가른다.
split함수를 생각하면 비슷할듯!

선언한 StringTokenizer에 nextToken() 함수를 쓰면 br.readLine()을 통해 입력받은 값을 공백단위로 구분해서 nextToken() 을 통해 순서대로 호출할 수 있다. 기본적으로는 공백이 구분자로 사용되지만, StringTokenizer 선언시 2번째 파라미터로 별도로 구분자를 추가할 수도 있다.

System.out.println()을 이용하는 방법!

print로도 println으로도 C언어에서 사용했던 printf로도 사용할 수 있다. C언어에 익숙한 사람이라면 printf로 사용해도 무방할 듯 하다.
